# Story 1.3 — Visualisation Live du Navigateur (Démo Publique)

## User Story

**En tant que** démonstrateur au hackathon,  
**Je veux** voir le navigateur Chromium naviguer en temps réel sur un écran pendant qu'un utilisateur parle au téléphone,  
**Afin que** les juges puissent voir visuellement l'agent IA piloter le web en direct — pas juste entendre une réponse vocale.

---

## Critères d'Acceptation

1. **Étant donné** qu'OpenClaw reçoit une tâche web, **quand** Chromium s'ouvre sur le VPS, **alors** on voit le navigateur naviguer en direct dans un onglet du navigateur local (`http://VPS_IP:6080`).
2. **Étant donné** que noVNC est démarré, **quand** j'ouvre `http://VPS_IP:6080` dans un navigateur, **alors** je vois le bureau virtuel du VPS avec Chromium visible.
3. **Étant donné** que l'agent reçoit la commande "Liste les dentistes sur Doctolib", **quand** OpenClaw navigue, **alors** on voit Doctolib s'ouvrir, les filtres s'appliquer et les résultats défiler en direct à l'écran.
4. **Étant donné** que noVNC est déployé, **quand** je tente de m'y connecter depuis internet sans mot de passe, **alors** l'accès est refusé (protection par token/mot de passe).

  Type=simple

- [x] **1.3.3** Créer le service systemd pour x11vnc (serveur VNC sur l'écran virtuel) :
  ```bash
  cat > /etc/systemd/system/x11vnc.service << 'EOF'
  [Unit]
  Description=VNC Server for Virtual Display
  After=xvfb.service
  Requires=xvfb.service

  [Service]
  Type=simple
  ExecStart=/usr/bin/x11vnc -display :99 -nopw -listen localhost -xkb -forever -shared
  Restart=always
  RestartSec=2

  [Install]
  WantedBy=multi-user.target
  EOF
  systemctl daemon-reload && systemctl enable x11vnc && systemctl start x11vnc
  ```
  **Résultat :** `active (running)` — VNC desktop sur localhost:5900 ✅

- [x] **1.3.4** Créer le service systemd pour noVNC (stream web sur port 6080) :
  ```bash
  NOVNC_PATH=$(find /usr -name "novnc_proxy" 2>/dev/null | head -1 || echo "/usr/bin/novnc_proxy")

  cat > /etc/systemd/system/novnc.service << 'EOF'
  [Unit]
  Description=noVNC Web VNC Client
  After=x11vnc.service
  Requires=x11vnc.service

  [Service]
  Type=simple
  ExecStart=/usr/bin/websockify --web=/usr/share/novnc 6080 localhost:5900
  Restart=always
  RestartSec=2

  [Install]
  WantedBy=multi-user.target
  EOF
  systemctl daemon-reload && systemctl enable novnc && systemctl start novnc
  ```
  **Résultat :** `active (running)` — websockify proxying :6080 → localhost:5900 ✅
  **Note :** websockify est le binaire utilisé (pas novnc_proxy), web root = `/usr/share/novnc`

## Notes Dev — Implémentation Story 1.3

- **`browser.display` invalide dans openclaw.json :** OpenClaw valide strictement son schema JSON. La clé `browser.display` n'est pas reconnue → le gateway refuse de démarrer avec `Unrecognized key: "display"`. La solution : injecter `DISPLAY=:99` via le drop-in systemd (`/etc/systemd/system/openclaw.service.d/display.conf`) — déjà en place depuis Story 1.2. Ne pas ajouter `display` dans openclaw.json.
- **`browser.headless = false` accepté :** Cette clé est reconnue par le schema OpenClaw et permet à Chromium de s'afficher sur le Xvfb.
- **Chemin Chromium :** Utiliser le binaire Playwright `/home/openclaw/.cache/ms-playwright/chromium-1208/chrome-linux64/chrome` avec `--no-sandbox --disable-dev-shm-usage` — pas `chromium-browser` (snap incompatible avec systemd, identifié en Story 1.2).
- **Erreurs dbus :** Non-critiques. Sur un VPS sans session D-Bus complète, Chromium log des erreurs `Failed to connect to the bus` mais fonctionne normalement.
- **websockify vs novnc_proxy :** Le package Ubuntu `novnc` fournit `/usr/bin/websockify` comme proxy. Les fichiers web sont dans `/usr/share/novnc/`. L'URL de démo : `http://<VPS_OU_DOMAINE>:6080/vnc.html`
- **Port 6080 :** Ouvert uniquement pour la démo — à fermer dès la fin (`ufw delete allow 6080/tcp`). Pour un accès permanent sécurisé, utiliser un tunnel SSH : `ssh -L 6080:localhost:6080 utilisateur@<VPS_OU_DOMAINE>`
- **Résolution 1920×1080 :** Idéale pour la démo sur grand écran. Réduire à 1280×720 si les performances VPS sont limitées
- **OpenClaw headless=false :** Ce paramètre est spécifique à la démo. En production (non-voyants), headless=true suffit — pas besoin d'affichage
- **`--no-sandbox` :** Requis pour Chromium en root (identifié en Story 1.1)
- **noVNC vs VNC client :** noVNC est web-based — les juges n'ont rien à installer, juste un navigateur
- **Mode SSH tunnel (alternatif sécurisé) :** `ssh -L 6080:localhost:6080 root@VPS_IP` puis `http://localhost:6080` — pas besoin d'ouvrir le port sur UFW
