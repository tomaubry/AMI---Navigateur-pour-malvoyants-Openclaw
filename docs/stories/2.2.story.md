# Story 2.2 — Pipeline STT : Deepgram Streaming (Audio → Texte)

## User Story

**En tant que** orchestrateur,
**Je veux** brancher Deepgram sur le stream audio Twilio pour transcrire la parole de l'utilisateur en texte en temps réel,
**Afin que** les mots de l'utilisateur soient disponibles en moins de 300ms pour le LLM.

---

## Critères d'Acceptation

1. **Étant donné** que l'utilisateur parle, **quand** Deepgram reçoit l'audio, **alors** les transcriptions partielles arrivent en < 300ms.
2. **Étant donné** un silence de 800ms, **quand** le VAD le détecte, **alors** la transcription finale est émise comme "phrase complète".
3. **Étant donné** une phrase transcrite, **quand** elle est émise, **alors** elle est loguée et transmise au LLM (story 2.5).

## Tâches

- [x] **2.2.1** Créer un compte Deepgram (deepgram.com) et récupérer la clé API :
  ```
  DEEPGRAM_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  ```

- [x] **2.2.2** Créer `src/stt.js` — module Deepgram streaming :
  ```javascript
  import { createClient, LiveTranscriptionEvents } from '@deepgram/sdk';

  export function createSTTStream(onTranscript) {
    const deepgram = createClient(process.env.DEEPGRAM_API_KEY);

    const connection = deepgram.listen.live({
      model: 'nova-2',
      language: 'fr',
      encoding: 'mulaw',      // format Twilio natif — pas de conversion !
      sample_rate: 8000,
      channels: 1,
      punctuate: true,
      interim_results: true,
      endpointing: 800,       // silence 800ms = fin de phrase
      utterance_end_ms: 1000,
    });

    connection.on(LiveTranscriptionEvents.Transcript, (data) => {
      const transcript = data.channel.alternatives[0].transcript;
      if (!transcript) return;

      if (data.is_final) {
        console.log('[STT] Final:', transcript);
        onTranscript(transcript);
      }
    });

    connection.on(LiveTranscriptionEvents.Error, (err) => {
      console.error('[STT] Erreur Deepgram:', err);
    });

    return {
      send: (audioChunk) => connection.send(audioChunk),
      close: () => connection.finish(),
    };
  }
  ```

- [x] **2.2.3** Intégrer le STT dans `src/server.js` :
  ```javascript
  import { createSTTStream } from './stt.js';

  wss.on('connection', (ws) => {
    const stt = createSTTStream((transcript) => {
      console.log('[Session] Transcript reçu:', transcript);
      // → sera transmis au LLM (story 2.5)
    });

    ws.on('message', (data) => {
      const msg = JSON.parse(data);
      if (msg.event === 'media') {
        // Décoder base64 et envoyer directement à Deepgram (mu-law 8kHz natif)
        const audioBuffer = Buffer.from(msg.media.payload, 'base64');
        stt.send(audioBuffer);
      }
      if (msg.event === 'stop') stt.close();
    });

    ws.on('close', () => stt.close());
  });
  ```

- [x] **2.2.4** Tester en appelant le numéro et en parlant :
  ```bash
  pm2 logs ami-orchestrator --lines 50
  # Doit montrer : [STT] Final: "bonjour je veux trouver un dentiste"
  ```

## Notes Dev

- **`encoding: 'linear16'`** : On utilise PCM 16kHz depuis `getUserMedia` navigateur (pas mu-law Twilio).
- **`nova-2` + `language: 'fr'`** : Meilleur modèle Deepgram pour le français. Latence ~200ms.
- **`endpointing: 800`** : Deepgram attend 800ms de silence pour émettre la transcription finale. Réduire à 500ms si l'agent semble trop lent à répondre.
- **Crédits Deepgram :** $200 de crédits gratuits à l'inscription — largement suffisant pour le hackathon.
- **Transcriptions partielles :** `interim_results: true` donne des résultats en temps réel. Utile pour les interruptions (story 3.1).
- **⚠️ SDK v5 bug auth WebSocket Node.js :** Le SDK `@deepgram/sdk` v5 utilise `Sec-WebSocket-Protocol` pour l'auth (conçu pour les navigateurs), ce qui échoue en Node.js avec une erreur "non-101 status code". **Solution implémentée :** bypass du SDK — connexion directe via le package `ws` avec header `Authorization: Token <key>`. URL : `wss://api.deepgram.com/v1/listen?...params`.
- **Buffering au démarrage :** `connect()` du SDK retourne avant que le WS soit open. On bufferise les premiers chunks audio et on les flush sur l'événement `open`.
